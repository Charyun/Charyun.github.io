常用命令

设置用户名和邮箱

  • 项目级别：
  • 系统用户级别： 原则：就近原则，优先项目级别，不过项目级别作用范围为本项目。项目级别的可以从.git路径下的config中查看到当前项目的用户信息；系统用户级别的在 ~/.gitconfig 文件中

暂存

使用 git add 将文件存入暂存区，使用如下命令进行撤销此次暂存

如果修改了暂存区中的文件可以使用 git add <file> 更新，或者使用 git restore <file> 撤销

提交

使用 git commit 进行提交

  • 如果修改了提交后的文件可以使用 git add <file> 更新，然后使用 git commit 再次提交，或者使用 git restore <file>丢弃该修改内容
  • 如果想撤销 commit，可以使用

查找历史记录

  • git log : 显示详细的历史commit记录
  • git log --pretty=onrline : 每一次commit 只显示一行, 或者使用 git log --oneline
  • git reflog : 可以查看HEAD指针移动的历史记录

reset 命令

  1. 基于索引值来指定HEAD的指向

  2. 使用 ^ : 只能往后退，往后退^的个数

  3. 使用 ~ 表示退n个

reset命令的其它两个参数

  • --soft : 只更新本地库的HEAD指针，不影响工作区和暂存区
  • --mixed : 只更新本地库和暂存区的HEAD指针，不影响工作区
